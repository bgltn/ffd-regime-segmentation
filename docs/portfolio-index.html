Abstract

This public release presents the methodological core of an FFD-based regime segmentation pipeline for macroeconomic release series. For each predefined break window, the framework asks whether a local differencing order estimated on regime data improves out-of-sample performance relative to a global benchmark estimated from pre-regime history.

A pair is retained only if both conditions hold: the local specification improves out-of-sample mean squared error by at least the public threshold, and the implied change in admissible differencing order is materially non-zero. Proprietary data, ingestion logic, and the full internal robustness layer are intentionally excluded from the public release.

Interpretation

The accepted set is heterogeneous in the sign of Δd. Some windows require larger local differencing orders, while others require smaller ones, indicating that the persistence shift is regime-specific rather than directionally uniform across series.

The strongest accepted case in the audited output is NAPMPMI / Eurozone / EurozoneW3 with an out-of-sample MSE improvement of 0.6047, while the weakest passing case is CPIX / LatinAmerica / LatinAmericaW1 with an improvement of 0.0909.

Method

1. Transformation

Map raw series to an economically meaningful scale before differencing.

2. Fixed-width FFD

Apply fixed-width fractional differencing using recursive binomial weights over a public admissible grid.

3. Admissibility

Select the smallest admissible order for which the ADF criterion rejects non-stationarity at the public level.

4. Regime validation

Use exogenous break windows, a pre-regime baseline, sparse release timing, and a chronological regime split.

5. Joint gate

Require both sufficient out-of-sample improvement and a material shift in differencing order.

6. Audit layer

Return explicit failure reasons and compact reliability warnings for estimable cases.

Gating logic

The segmentation label is determined by a joint quantitative rule. Admissibility via ADF defines feasible differencing orders, but acceptance depends on the out-of-sample comparison and the materiality threshold on Δd. 

SEGMENTED iff
MSE_local <= (1 - ETA) * MSE_global
and
|d_local - d_global| >= TAU

Public defaults are ADF alpha = 5%, train split = 80%, minimum baseline length = 6, minimum train length = 8, minimum test length = 6, ETA = 0.05, and TAU = 0.05.

Reliability logic

The public reliability layer is a compact warning system. It distinguishes stronger accepted cases from potentially fragile cases without claiming formal uncertainty quantification.

boundary

At least one estimated differencing order lies at the edge of the admissible public grid.

sample_bind

The effective training or test support is at the public minimum floor imposed by the chronological split.

stability

The implied persistence shift is large enough to warrant caution in interpretation.

Note: this page describes only the reliability warnings generated by the public code.

Limitations

  • The page reports accepted empirical cases, not predictive generality across all macro series.
  • The public reliability layer is not formal uncertainty quantification.
  • The released code omits proprietary data, ingestion logic, and the full internal robustness stack.
  • The workflow uses predefined exogenous regimes rather than estimating endogenous breakpoints.

Reproducibility

The public repository is reproducible at the level of methodological logic on user-supplied arrays, but it is not plug-and-run for arbitrary series names. Before calling validate_regime(...), the user must register the series in LEVEL_SERIES or LOG_SERIES because the transformation layer depends on the public series registry. 

import numpy as np
import core_pipeline as cp

cp.LEVEL_SERIES = frozenset({"YOUR_SERIES"})
# Alternatively, register the series in cp.LOG_SERIES if log transformation is appropriate.

before = np.array([...], dtype=float)
during = np.array([...], dtype=float)

result = cp.validate_regime(
    series_name="YOUR_SERIES",
    before_values=before,
    during_values=during,
    pair_id="YOUR_SERIES|Window_1",
)
