Released algorithm specification

Fractional Difference Regime Segmentation

Confirmation algorithm for exogenous break regimes. The released procedure tests whether a pre-regime admissible fractional differencing order remains valid inside a known macroeconomic regime window.

Notation

Symbol Definition
x_t Observed macroeconomic release series in point-in-time form
D^d(x) Fixed-width fractional difference of order d
d*global Smallest admissible fractional differencing order estimated on pre-regime data
d*local Smallest admissible fractional differencing order estimated on the regime training sample
Δd d*local − d*global
MSEglobal Out-of-sample mean squared error under the global operator
MSElocal Out-of-sample mean squared error under the local operator
η Minimum out-of-sample improvement threshold; released value 0.05
τ Minimum separation threshold for |Δd|; released value 0.05

Empirical question and regime design

Given a known regime change at time TR, does the data-generating process of a macroeconomic series require a different long-memory parameter to achieve the same admissible level of stationarity and out-of-sample generalisation inside the regime as outside it?

Regime boundaries are not inferred from the series. They are specified a priori from documented economic events and institutional changes and treated as part of the data-generating process.

This separation avoids circularity between regime detection and regime confirmation. All series are used in point-in-time form: the baseline is strictly pre-window, there is no forward fill, and right-censored windows at the analysis horizon are explicitly flagged.

Released procedure

  1. Choose a series x_t and an exogenous regime window R = [TR, TE].
  2. Construct the pre-regime baseline sample using observations strictly before TR.
  3. Split the regime window chronologically into a local training segment and a local test segment.
  4. Map the series to the released level or log scale according to the public registry.
  5. For each candidate d on the released admissible grid, compute D^d(x) using fixed-width fractional differencing.
  6. Apply the released ADF admissibility rule and select the smallest admissible order on the baseline sample. This yields d*global.
  7. Apply the same rule on the regime training sample and select the smallest admissible order. This yields d*local.
  8. Transform the regime data using both operators and evaluate out-of-sample performance with the released AR(2) rolling scoring mechanism.
  9. Compute Δd = d*local − d*global.
  10. Label the pair SEGMENTED only if both released gates hold jointly.

Decision gate

SEGMENTED iff

MSE_local ≤ (1 - η) · MSE_global
and
|Δd| ≥ τ

Released values:
η = 0.05
τ = 0.05

This dual gate enforces predictive relevance and material separation in the memory parameter.

Interpretation of Δd

  • Δd > 0: the regime requires more differencing than the baseline estimate.
  • Δd < 0: the regime requires less differencing than the baseline estimate.
  • Under fractional differencing, smaller d implies greater preservation of low-frequency dependence.

Conclusion

  • Regime-conditional memory under exogenous breaks. Economic chronologies define exogenous windows. The pipeline tests whether these windows are associated with changes in long-memory behaviour that justify a regime-specific differencing order rather than a single global operator.
  • Confirmation test for d-invariance. ADF defines the feasible region for d. Out-of-sample generalisation error under a common AR(2) scoring rule is the validation functional. The choice between global and local fractional differencing is empirical and falsifiable.
  • Structured diagnostic output. Regime-level tables and compact reliability flags show where a local differencing order is beneficial, where it is fragile, and where the global operator remains adequate.

For a non-trivial subset of macroeconomic series and regimes, regime-specific fractional differencing yields lower out-of-sample error than a global differencing order under the same stationarity constraint. For the remainder, the global operator is sufficient. The result is conditional, not universal.
