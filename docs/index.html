FFD Regime Segmentation

This site presents the public methodological core of a regime-conditional fractional differencing pipeline for macroeconomic release series.

The central question is whether the minimal admissible differencing order remains stable across exogenously defined regime windows, or whether a regime-specific estimate reduces out-of-sample loss under a causal split.

Public result. Out of 168 tested series-window pairs, 10 passed the public admissibility, out-of-sample, and segmentation gates.

This release is a research-facing public layer. It exposes the method, the decision rule, selected accepted empirical cases, and a compact reliability register. It does not expose private data, private file paths, internal notebooks, or the full internal robustness stack.

Question and scope

The public question is narrow. For each predefined regime window, the framework asks whether a local fractional differencing order, estimated on regime data, improves out-of-sample performance relative to a global benchmark estimated from pre-regime history.

The release is limited to the methodological core on user-supplied arrays, selected accepted empirical cases, and a compact audit layer. It is not a production platform, not a full replication package, and not a claim of predictive generality across all macroeconomic series.

Public implementation

The public code exposes the released decision logic in six stages. The sequence below mirrors the methodological structure of the repository.

Stage 1

Transformation

Map the raw series to the public level or log scale before differencing.

Stage 2

Fixed-width FFD

Apply truncated fractional differencing on a public admissible grid.

Stage 3

Admissibility

Select the smallest admissible order satisfying the public ADF rule.

Stage 4

Chronological validation

Compare baseline and local specifications using a causal regime split.

Stage 5

Decision

Require both predictive gain and a material shift in differencing order.

Stage 6

Audit

Return explicit failure reasons and compact reliability warnings.

Worked example

One regime-window pair is processed through the same public stages used by the released code. The purpose of this section is to show the mechanics of the implementation, not to disclose private data.

Input
The user supplies two arrays: before_values for the pre-regime baseline and during_values for the regime window.
Transformation
The series is mapped to the public level or log scale, according to the released registry logic.
FFD grid
Fixed-width fractional differences are computed over the public admissible grid. Each candidate order is tested only within the released gate structure.
Admissible orders
The pipeline selects the smallest admissible global order from the baseline data and the smallest admissible local order from the regime training subset.
Validation
A chronological split compares the local and global transformed series out of sample. The public comparison metric is mean squared error.
Decision
The pair is labelled SEGMENTED only if the local specification delivers sufficient out-of-sample improvement and the change in differencing order is materially non-zero.

Interpretation

The accepted set is heterogeneous in the sign of Δd. Some windows require larger local differencing orders, while others require smaller ones. That pattern supports a regime-conditional interpretation rather than a single global persistence adjustment.

The strongest accepted case in the public table is NAPMPMI / Eurozone / EurozoneW3 with out-of-sample MSE improvement of 0.6047. The weakest passing case is CPIX / LatinAmerica / LatinAmericaW1 with improvement of 0.0909.

Decision rule

A pair is labelled segmented only under a joint quantitative rule. Admissibility defines feasible differencing orders, but acceptance also requires predictive gain and material memory shift. 

SEGMENTED iff
MSE_local <= (1 - ETA) * MSE_global
and
|d_local - d_global| >= TAU

Public defaults are: ADF alpha = 5%, train split = 80%, minimum baseline length = 6, minimum train length = 8, minimum test length = 6, ETA = 0.05, and TAU = 0.05.

Reliability logic

The public reliability layer is a compact warning system. It separates stronger accepted cases from potentially fragile accepted cases without claiming formal uncertainty quantification.

boundary

At least one estimated differencing order lies at the edge of the public admissible grid.

sample_bind

The effective training or test support is at the public minimum floor imposed by the chronological split.

stability

The implied persistence shift is large enough to warrant caution in interpretation.

This site describes only the compact reliability warnings generated by the released public layer.

Public boundary

  • Public: methodological core, released decision logic, selected accepted empirical cases, and compact reliability flags.
  • Omitted: proprietary data, private paths, internal notebook chains, private robustness infrastructure, and study-specific private registries.
  • Interpretation limit: accepted cases do not establish predictive generality across all macroeconomic series.

Limitations

  • The public site reports accepted empirical cases, not full discovery output.
  • The public reliability layer is not formal uncertainty quantification.
  • The workflow relies on predefined exogenous regime windows rather than estimating endogenous breakpoints.
  • The released code is a methodological companion, not the private research system end to end.

Reproducibility

The public repository is reproducible at the level of methodological logic on user-supplied arrays. Repository-level source code remains in the src/ layer rather than inside the site directory.

The public module is src/core_pipeline.py. The snippet below shows how to import and call the pipeline on user-supplied arrays. 

import numpy as np
import src.core_pipeline as cp

cp.LEVEL_SERIES = frozenset({"YOUR_SERIES"})
# Alternatively, register the series in cp.LOG_SERIES if log transformation is appropriate.

before = np.array([...], dtype=float)
during = np.array([...], dtype=float)

result = cp.validate_regime(
    series_name="YOUR_SERIES",
    before_values=before,
    during_values=during,
    pair_id="YOUR_SERIES|Window_1",
)

print(result)
