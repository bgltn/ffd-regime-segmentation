Algorithm Specification

FFD-based regime segmentation of macroeconomic release series.

This document describes a protected public version of the pipeline used to test whether the memory structure of a macroeconomic series changes across an exogenous regime window.

Notation

SymbolDefinition

x̃tTransformed series: log for index-like variables, levels for rate-like variables
dFractional differencing order on an admissible grid in [0, 1]
FFDd(x̃)Fixed-width fractionally differenced series
d*Minimal admissible differencing order under an ADF criterion
dglobalMinimal admissible order estimated on the baseline sample
dlocalMinimal admissible order estimated on the regime-training sample
Δddlocal − dglobal

Pipeline

1. Transformation

Each raw series is first mapped to an economically meaningful scale. For index-like series, the transformation is the natural logarithm of xt. For rate-like series, the transformation is the identity, so the series is left in levels. This transformation is applied before fractional differencing.

2. Fixed-width fractional differencing

The transformed series is filtered using the fixed-width fractional differencing operator. The recursive binomial weights are defined as:

w_0 = 1
for k ≥ 1:
    w_k = -w_(k-1) × (d - k + 1) / k

The sequence of weights is truncated using a tolerance parameter. That truncation determines the effective width of the fixed-width filter.

3. Minimal admissible order

For each candidate order on a coarse admissible grid in [0, 1]:

  • Apply FFDd to the transformed series.
  • Run the ADF test on the filtered output.
  • Return the smallest d that satisfies the stationarity criterion.

If no candidate order satisfies the criterion, the pair is marked inadmissible at this stage.

4. Regime validation

For a (series, regime-window) pair:

  • Apply pre-gates for minimum sample size.
  • Transform the baseline and regime samples.
  • Split the regime sample chronologically into training and test partitions.
  • Estimate dglobal on the baseline.
  • Estimate dlocal on the regime-training partition.
  • Score both orders out of sample on the regime test partition.
  • Compare predictive performance and |Δd|.

5. Segmentation decision

A pair is labelled SEGMENTED only if:

  • The local order improves out-of-sample predictive loss relative to the baseline order by at least a prespecified threshold.
  • The absolute change in differencing order exceeds a materiality threshold.

Exact calibration belongs to the study design and can be stated in the paper without embedding the full internal calibration layer in public code.

Public audit layer

Failure register

Every non-segmented or infeasible pair receives a failure category, such as:

  • Baseline too short
  • Regime sample too short
  • Transform error
  • ADF infeasible
  • OOS score undefined
  • Predictive gain below threshold
  • Memory shift below threshold

Reliability register

The public reliability register is intentionally compact. It reports only the warnings implemented in the public code:

  • boundary — at least one estimated differencing order lies at the edge of the admissible public grid.
  • sample_bind — the effective training or test support is at the public minimum floor imposed by the chronological split.
  • stability — the absolute persistence shift |Δd| exceeds a high-threshold cutoff, so interpretation should be cautious.

More detailed internal heuristics may be retained privately.
